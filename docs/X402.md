# x402 Payment Discovery

Disruption Intelligence uses x402 for paid API access. Developers and agents can discover payment requirements directly from HTTP responses without obtaining a conventional API key.

## Discovery Flow

1. Read the hosted API contract or x402 manifest.
2. Send a request to a paid endpoint.
3. If no payment authorization is supplied, receive `402 Payment Required` and active payment metadata.
4. Use an x402-capable client and funded wallet to authorize the required payment.
5. Retry the request with the authorization generated by the client.

## Inspect a Challenge Without Paying

The following request only inspects the challenge when sent without payment authorization:

```bash
curl -i "https://warn.forgemesh.io/territory/77002/disruption?radius=50"
```

Inspect public discovery documents:

```bash
curl "https://warn.forgemesh.io/.well-known/x402.json"
curl "https://warn.forgemesh.io/openapi.json"
curl "https://warn.forgemesh.io/llms.txt"
```

## Payment Terms

- Each priced route identifies its listed price in the discovery contract.
- The live `402` response provides the active network, receiving address, and authorization requirements.
- A client should not hardcode a receiving address or rely on a copied manifest.
- Completing a paid request transfers funds according to the live payment terms.
